# Pytest Guide

## Overview

This project uses the following libraries for testing:

- pytest
- pytest-asyncio
- pytest-mock

Spider tests do not send real HTTP requests or launch a browser. Playwright is fully mocked, allowing tests to run quickly and deterministically.

---

## Running Tests

Run all tests:

```bash
pytest
```

Run with detailed output:

```bash
pytest -v
```

Run a specific file:

```bash
pytest tests/test_search_for_coin.py
```

Run a specific test:

```bash
pytest tests/test_search_for_coin.py::test_parse
```

## Testing Async Code

Spider methods are asynchronous, so async tests use:

```python
@pytest.mark.asyncio
```

The first request generated by `start()` is retrieved with:

```python
request = await anext(spider.start())
```

Async generators returned by `parse()` are collected with:

```python
items = [item async for item in spider.parse(response)]
```

---

## Mocking Playwright

Playwright is mocked using `pytest-mock` and `AsyncMock`.

Example:

```python
page = mocker.Mock()
page.content = AsyncMock(return_value=html)
page.close = AsyncMock()
```

A fake `HtmlResponse` is then created and the mocked page is attached to:

```python
response.meta["playwright_page"]
```

This simulates the behavior of Scrapy Playwright without launching a browser.

---

## Mocking Browser Elements

Browser interactions are mocked by replacing Playwright objects.

Examples include:

- `page.locator()`
- `page.get_by_role()`
- `wait_for()`
- `fill()`
- `click()`

`side_effect` is used when multiple locator calls should return different mocked objects.

---

## Assertions

The tests verify:

- Spider initialization
- Input validation
- Raised exceptions
- Generated requests
- Parsed items
- Browser interactions
- Resource cleanup

Examples:

```python
assert_called_once_with(...)
assert_awaited_once()
assert_awaited_once_with(...)
```

---

## Exception Testing

Expected exceptions are verified using:

```python
with pytest.raises(ValueError):
    ...
```

When the error message is also important:

```python
with pytest.raises(ValueError, match="Coin not found"):
    ...
```

---

## Parameterized Tests

Multiple inputs can be tested with a single function:

```python
@pytest.mark.parametrize(...)
```

This avoids duplicated test code and improves readability.

---

## Patching Functions

External dependencies are replaced using:

```python
mocker.patch(...)
```

Always patch the object where it is imported, not where it is originally defined.

---

## Utility Tests

Utility functions are tested directly using normal assertions.

Example:

```python
assert get_url_by_symbol("BTC") == "/currencies/bitcoin/"
```

No mocking is required unless the function depends on external resources.

---

## Adding a New Spider Test

When creating a new Spider test:

1. Save the target HTML inside `tests/data/`.
2. Mock the Playwright page.
3. Create a fake `HtmlResponse`.
4. Attach the mocked page to `response.meta`.
5. Execute the spider.
6. Verify the parsed data.
7. Verify Playwright interactions.
8. Verify that `page.close()` is always awaited.
